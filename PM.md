# 投资决策适配检测 — 开发指南（详细版）

## 这个产品是什么

帮投资者在买东西之前，检查这个决定适不适合自己。系统根据用户的真实交易行为（而不只是问卷自评）来判断，输出 Fit / Caution / Mismatch 三个等级，加 AI 生成的解释。

不推荐买什么，只做检查。

## 核心用户流程

### 首次使用（建立基线）

1. 注册登录
2. 完成简短自评（风险等级 R1-R5、投资期限、最大可接受亏损）
3. 连接券商账户，系统读取历史交易记录（demo 用预置模拟数据）
4. 系统生成行为画像：自评 vs 实际行为的对比（雷达图）

### 日常使用（决策前检测）

5. 首页展示近期热门产品，用户也可以手动输入想买的产品（输入产品名或代码）
6. 用户点击或输入后，系统跑 fitness check
7. 输出检测结果 + AI 解释 + 反思提示
8. 用户自行决定买或不买

### 持续追踪

9. 系统同步新的交易数据，行为画像持续更新

### 第一层：数据与规则引擎（SQL / API）

**做什么：** 从交易数据里算出用户的真实行为特征，跟自评对比，判断有没有错配。

**4 个核心行为信号及计算逻辑：**

**信号 1：持有期偏差**

```
输入：investors.stated_horizon + transactions.hold_days (action = 'sell')
计算：该用户所有卖出记录的中位持有天数
对比：自述期限对应的合理范围
    <6m → 合理范围 30-180 天
    6m-1y → 合理范围 120-365 天
    1-3y → 合理范围 200-1000 天
    3-5y → 合理范围 500-1800 天
    5y+ → 合理范围 800 天以上
判断：实际中位数低于合理范围下限 → 标记偏差
```

**信号 2：恐慌卖出倾向**

```
输入：transactions (action = 'sell', market_change_pct < 0)
计算：亏损卖出中，market_change_pct 在 -10% 到 0% 之间（小跌就跑）的比例
判断：
    比例 > 50% → 高恐慌倾向
    比例 25-50% → 中等
    比例 < 25% → 低
同时看 sell_decision_source 中 panic 的占比作为辅助确认
```

**信号 3：外部依赖度**

```
输入：transactions.decision_source (action = 'buy')
计算：decision_source 为 social_media 或 friend 的买入占总买入的比例
判断：
    比例 > 50% → 高外部依赖
    比例 25-50% → 中等
    比例 < 25% → 低
```

**信号 4：流动性冲突**

```
输入：investors.has_short_term_cash_need + transactions (action = 'buy')
计算：has_short_term_cash_need = true 的用户，是否还在买入非流动性产品（如 fund、private_equity 等）
判断：有短期用钱需求 + 近期买入了锁定期较长的产品 → 标记冲突
```

**输出格式（JSON 示例）：**

```json
{
  "investor_id": "INV0042",
  "signals": {
    "holding_deviation": {
      "stated": "3-5y",
      "actual_median_days": 47,
      "level": "high",
      "detail": "自述 3-5 年，实际中位持有 47 天"
    },
    "panic_sell": {
      "rate": 0.62,
      "level": "high",
      "detail": "62% 的亏损卖出发生在跌不到 10% 时"
    },
    "external_dependency": {
      "rate": 0.45,
      "level": "medium",
      "detail": "45% 的买入受社交媒体或朋友影响"
    },
    "liquidity_conflict": {
      "has_conflict": true,
      "detail": "近 6 个月有大额支出需求，但仍在买入基金类产品"
    }
  },
  "fitness_result": "likely_mismatch",
  "fitness_score": 32,
  "triggered_signals": 3
}
```

**检测结果判定规则：**

| 结果            | 触发条件                                                     |
| --------------- | ------------------------------------------------------------ |
| Fit             | 4 个信号都没有 high 级别                                     |
| Caution         | 1-2 个信号为 high，或 2 个以上为 medium                      |
| Likely Mismatch | 3 个以上信号为 high，或任意信号严重偏离（如持有期偏差超过 10 倍） |

**Fitness Score 计算（0-100）：**

```
每个信号满分 25 分
high = 0 分, medium = 15 分, low = 25 分
总分 = 四个信号分数之和
```

**Taya 的工作：** SQL 查询逻辑已验证，阈值和判定规则由 Taya 定义。
**Eric 的工作：** 把这些逻辑写成后端 API，接收 investor_id + 投资意图，返回 JSON 结果。

### 第二层：RAG 知识检索（详细实现）

**做什么：** 当第一层检测到某个信号被触发时，去知识库里找到对应的解释内容，传给第三层 LLM 做参考。

**为什么需要 RAG：** 如果不用 RAG，LLM 只能泛泛地说"你可能有恐慌卖出倾向"。有了 RAG，LLM 可以引用专业的行为金融学解释，让回答更可信。

**知识库内容（Taya 撰写，预计 20-30 篇短文）：**

分两类：

**A 类：行为金融学解释（面向 B2C 用户解释）**

| 对应信号   | 知识条目示例                                                 |
| ---------- | ------------------------------------------------------------ |
| 持有期偏差 | "为什么我们说要长期持有，却总是提前卖出？"——时间偏好不一致、近因效应、损失厌恶 |
| 持有期偏差 | "短线操作为什么往往跑不赢长期持有？"——交易成本、择时困难、税务影响 |
| 恐慌卖出   | "什么是短视损失厌恶？"——人对亏损的痛感是盈利快感的约 2 倍    |
| 恐慌卖出   | "小跌就卖和真正的止损有什么区别？"——止损是事前计划、恐慌是事后反应 |
| 外部依赖   | "跟风投资为什么危险？"——信息不对称、别人的风险承受力不是你的 |
| 外部依赖   | "社交媒体上的投资建议为什么不可靠？"——幸存者偏差、选择性展示 |
| 流动性冲突 | "为什么要留出应急资金再投资？"——流动性需求和投资期限的基本原则 |
| 流动性冲突 | "锁定期产品适合什么样的人？"——资金规划的基本框架             |
| 通用       | "风险承受能力和风险承受意愿的区别"——能力是客观的、意愿是主观的 |
| 通用       | "处置效应：为什么我们总是卖掉赚钱的、死拿亏钱的？"           |

**B 类：顾问沟通话术（面向 B2B 顾问模式）**

| 场景           | 知识条目示例                                                 |
| -------------- | ------------------------------------------------------------ |
| 客户高估自己   | "如何跟自评激进但实际保守的客户沟通？"——先肯定、再用数据引导、不要直接否定 |
| 客户低估自己   | "如何帮助过度保守的客户认识自己的真实承受力？"——从小金额尝试开始 |
| 客户预期不合理 | "客户觉得低风险产品赚太少怎么办？"——重新校准预期、对比无风险利率 |
| 客户跟风       | "客户说朋友推荐的一定要买怎么办？"——引导回到自身情况分析     |

**每篇知识文本格式：**

```
标题：什么是短视损失厌恶？
标签：panic_sell, loss_aversion, behavioral_finance
内容：（200-500 字的解释，用大白话写，不要学术腔）
来源参考：Kahneman & Tversky, 1979; Benartzi & Thaler, 1995
```

**RAG 技术实现步骤（Eric 做）：**

```
1. Taya 写好知识库内容 → 存成文本文件或 JSON
2. 用 embedding 模型把每篇文本变成向量
   推荐：OpenAI text-embedding-3-small（便宜够用）
3. 存入向量数据库
   简单方案：FAISS（本地，免费）
   云方案：Pinecone / Supabase pgvector
4. 检索时：
   输入 = 第一层触发的信号名称（如 "panic_sell"）
   输出 = 最相关的 2-3 段知识文本
5. 把检索到的文本传给第三层 LLM
```

**检索逻辑示例：**

```
第一层输出：panic_sell = high
→ 用 "恐慌卖出 损失厌恶 小跌就卖" 作为检索 query
→ 向量数据库返回最相关的 2 段文本
→ 传给第三层
```

**Taya 的工作：** 撰写知识库内容（20-30 篇），定义每篇的标签和对应信号。
**Eric 的工作：** 实现向量化、存储、检索的技术流程。

### 第三层：AI 解释生成（LLM）

**做什么：** 把第一层的数字和第二层的知识，组合成用户能看懂的个性化解释。

**Prompt 模板（Taya 设计）：**

```
你是一个投资决策适配检测助手。你的任务是用清晰、友善的语言，
帮助投资者理解他们当前的投资决定可能存在的风险认知偏差。

## 规则
- 绝对不要给出具体的买卖建议
- 绝对不要预测市场走势或收益
- 引用用户的具体历史行为数据作为证据
- 生成 2-3 个用户应该问自己的反思问题
- 语气友善但直接，不要居高临下

## 用户当前想做的事
{user_intent}
例如："想买一只 R4 级别的混合基金，打算持有 1 年以上"

## 第一层分析结果（来自 SQL）
{signal_results}
例如：
- 持有期偏差：自述 3-5 年，实际中位持有 47 天（high）
- 恐慌卖出倾向：62% 的亏损卖出发生在跌不到 10% 时（high）
- 外部依赖度：45% 的买入受外部影响（medium）
- 流动性冲突：无

## 相关知识（来自 RAG 检索）
{rag_context}
例如：
"短视损失厌恶是指投资者对短期亏损的痛感远大于对同等盈利的快感..."

## 检测结果
{fitness_result}
例如：Likely Mismatch, Score: 32/100

## 请生成
1. 一段 3-5 句话的个性化解释，引用用户的具体历史行为
2. 2-3 个反思问题
3. 置信度说明
```

**LLM 输出示例：**

```
你说你打算持有这只基金 1 年以上，但你过去半年买过 5 次类似产品，
其中 4 次在跌不到 8% 的时候就卖了，平均只拿了 47 天。
这说明你实际的持有耐心可能比你想的要短。
另外你有将近一半的买入是受朋友或社交媒体影响的，
这次买入的决定也建议你确认一下是自己分析过的还是跟风。

你可以问自己：
1. 如果这只基金下个月跌了 10%，我真的会继续拿着吗？
2. 我是因为自己研究过才想买，还是因为最近看到别人在聊？
3. 这笔钱我未来半年会不会需要用到？

本分析基于你过去 6 个月的交易数据，置信度为中等。
随着使用时间增加，分析会更精准。
```

**Taya 的工作：** 设计 prompt 模板、定义输出格式和语气要求、测试不同场景下的输出质量。
**Eric 的工作：** 接入 LLM API（推荐 OpenAI GPT-4o 或同等），组装 prompt，处理返回结果。

### 第四层：Guardrails（AI 边界与风控）

**做什么：** 确保 AI 输出不越界。

**硬性规则：**

| 规则           | 实现方式                                                     |
| -------------- | ------------------------------------------------------------ |
| 禁止买卖建议   | Prompt 里反复强调 + 输出后关键词检查（"建议买入""推荐""应该买"）→ 命中则拦截重新生成 |
| 禁止收益预测   | 同上（"会涨""预计收益""回报率"）                             |
| 禁止确定性判断 | 禁止说"你一定会亏"之类的话，只说"可能""历史数据显示"         |
| 置信度透明     | 每次输出必须附带数据基础说明（"基于 X 个月数据"）            |
| 用户可控       | 用户可以查看自己的行为画像、质疑标记、重置画像               |

**关键词黑名单（输出检查用）：**

```
买卖建议类：建议买入, 推荐购买, 应该买, 建议卖出, 赶紧买, 别买了
收益预测类：会涨, 会跌, 预计收益, 年化回报, 肯定赚, 稳赚
确定性判断类：你一定, 肯定会, 必然, 保证
```

**如果命中黑名单：** 不展示给用户，重新生成一次。连续 3 次命中则返回默认安全文案。

**Taya 的工作：** 定义黑名单关键词、定义安全文案兜底方案、验收测试时重点检查边界情况。
**Eric 的工作：** 实现输出过滤逻辑。

## 雷达图五个维度

| 维度                                | 含义                       | 数据映射                                    |
| ----------------------------------- | -------------------------- | ------------------------------------------- |
| Risk Tolerance（风险承受力）        | 自评 vs 实际能接受多大波动 | self_risk_level vs actual_tolerance         |
| Holding Patience（持有耐心）        | 说拿多久 vs 实际拿多久     | stated_horizon vs median hold_days          |
| Decision Independence（决策独立性） | 多少决策是自己做的         | decision_source 中 self 的占比              |
| Volatility Comfort（波动适应度）    | 跌的时候慌不慌             | panic sell rate + drawdown reaction         |
| Liquidity Readiness（流动性准备度） | 有没有短期用钱压力         | has_short_term_cash_need + 近期买入产品类型 |

蓝色 = 自评画像，橙色 = 行为推断画像。差距 = 错配。

---

## 检测结果三个等级

| 等级            | 含义                       | 触发条件                      | 界面表现           |
| --------------- | -------------------------- | ----------------------------- | ------------------ |
| Fit             | 这个决策跟你的行为模式一致 | 4 个信号都没有 high           | 绿色，简短确认     |
| Caution         | 部分信号有错配             | 1-2 个信号为 high             | 黄色，展开具体标记 |
| Likely Mismatch | 大概率不适合               | 3+ 信号为 high 或单个严重偏离 | 红色，AI 详细解释  |
